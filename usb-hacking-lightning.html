USB Hacking

Warum USB?

  • Sehr verbreitet: von Mäusen und Tastaturen über Soundkarten, Kameras und Keyboards
  • Sehr hackbar: gesamter Datenverkehr vom Host koordiniert und von dort mitzulesen
  • Viele Geräte nur mit Windows Treiber

So geht's

  1. Device-Id am Bus identifizieren: durch Vendor ID/Product ID oder Datenverkehr
  2. Gerät im regulären Betrieb beobachten und Daten aufzeichnen
  3. Grübeln
  4. Programmieren
  5. 3. & 4. nach Bedarf wiederholen

Terminologie

  • Unterteilung in Host (PC) und Gerät
  • Geräte identifiziert durch
    1. Device Descriptor: eindeutig für Gerät
    2. Configuration Descriptor: 1+ pro Gerät
    3. Interface Descriptor: innerhalb einer Konfiguration
    4. Endpoint Descriptor: repräsentiert logische Einheit innerhalb eines Geräts. Endpunkt 0 vorgegeben für Control Transfers.

4 verschiedene Arten der Datenübertragung

  • Control: kurze Befehle zur Steuerung eines Gerätes
  • Interrupt: niedrige Latenz, periodisch gepolled
  • Isochronous: garantierte Bandbreite, Paketverlust möglich
  • Bulk: Transfer grösserer Datenmengen mit Fehlerkorrektur
